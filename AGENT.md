# SentinelScan AI Development Rules

This repository contains the complete SentinelScan project.

Before implementing any feature, read the project context in the following order:

1. context/project-overview.md
2. context/architecture.md
3. context/code-standards.md
4. context/progress-tracker.md

---

## Core Principles

- Implement only the requested feature.
- Follow the documented architecture.
- Keep the project modular.
- Do not modify unrelated files.
- Prefer reusable code over duplication.
- Follow the existing project structure.
- Do not introduce unnecessary dependencies.
- Update `context/progress-tracker.md` after completing a meaningful feature.
- If architecture changes, update `context/architecture.md`.
- Keep documentation synchronized with implementation.

---

## AI Skills

Use the available skills when appropriate:

- `.agent/architect.md`
- `.agent/imprint.md`

## Code Quality

- Keep code simple.
- Prefer readability over cleverness.
- Follow existing naming conventions.
- Write self-explanatory code.
- Remove unused code immediately.
- Never leave TODOs without explanation.
